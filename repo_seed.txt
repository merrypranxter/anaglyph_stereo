# anaglyph_stereo

> stereoscopic illusions. anaglyph encodes left/right views in opposing color channels for red-cyan glasses; lenticular interlaces N views into strips that flip/ripple with viewing angle (simulated via mouse/time tilt). plus the no-glasses wiggle-3D.

**Type:** FINISHER (needs a depth map вЂ” generated or uploaded).

**Adjacency:** the `wiggle_parallax` mode overlaps `parallax_depth_fields`. Keep *this* repo's identity on **glasses-anaglyph + angular-lenticular**; treat wiggle as a minor bonus and let `parallax_depth_fields` own pure parallax.

## The Math

depth-based view shift: left/right views are generated by horizontal UV shift proportional to depth. `viewL = sample(src, uv + depth * sep * 0.5)`, `viewR = sample(src, uv - depth * sep * 0.5)`. the separation `sep` is the eye distance in UV space. the depth map drives the parallax.

anaglyph-dubois: optimized matrix anaglyph. the Dubois matrices reduce ghosting vs naive channel split. the matrices are in the docs. the red channel gets left-view weighted, cyan gets right-view weighted. it is a matrix multiplication per pixel, not a naive mask.

lenticular: `N` views interlaced into vertical strips of period `p`. reveal one view per strip by viewing angle `t`: `viewIndex = floor(uv.x / p + t) mod N`. the angle is simulated by mouse position or time sweep. the screen cannot physically tilt, so we fake it.

wiggle-parallax: 2-view oscillation at ~6Hz. the left and right views alternate. no glasses needed. the retro 3D-GIF look. it is a bonus mode, not the main identity.

## The Art

**red_cyan_classic** вЂ” Dubois anaglyph, separation = 0.03, depth_scale = 1.0. the image is for glasses. the ghosting is minimized. the depth is convincing. the palette is constrained by the medium: red channel carries left, cyan carries right. the image is a memory from a 3D comic book.

**lenticular_ripple** вЂ” `N=8` views, strip period `p = 1/120`, angle driven by mouse X. the image shimmers as you move the mouse. it is a holographic postcard: the depth ripples like water. `N=8` is enough for smooth motion; `N=4` is the budget version. the angle drive is the toy.

**wiggle_gif** вЂ” 2-view, 6Hz oscillation, no glasses. the image wobbles. the depth is perceived by the brain's motion parallax. it is crude, it is retro, it is universally viewable. `separation = 0.04`, `oscillation_speed = 6.0` Hz. the wiggle is the charm.

**depth_glitch** вЂ” exaggerated separation = 0.15, depth_scale = 2.0, channel tearing. the left and right views are so far apart that the image tears. the red channel is a ghost, the cyan is a corpse. it is an artifact aesthetic, not a stereo effect. the glitch is the point.

## Engines (Modes)

- **anaglyph-dubois** вЂ” optimized matrix anaglyph, least ghosting (Dubois matrices in docs).
- **anaglyph-color** вЂ” naive red/cyan.
- **lenticular-flip** вЂ” interlace `N` views into vertical strips; reveal by viewing angle.
- **wiggle-parallax** вЂ” 2-view ~6Hz oscillation, no glasses (retro 3D-GIF).

## Pipeline

1. depth pass (or uploaded depth).
2. **views.frag** вЂ” generate L/R (or N) views by depth-proportional horizontal shift.
3. **anaglyph.frag** (channel matrix) **or** **lenticular.frag** (strip select by angle).
4. **post** вЂ” slight ghost-reduction blur, chromatic.

## Parameters

- `separation`: 0.0вЂ“0.2 (eye distance in UV)
- `depth_scale`: 0.0вЂ“2.0 (parallax depth multiplier)
- `mode`: anaglyph_dubois | anaglyph_color | lenticular | wiggle
- `view_count`: 2вЂ“16 (for lenticular)
- `angle_drive`: mouse | time | gyro

## Palette

Anaglyph constrained to red/cyan; lenticular/wiggle carry the source palette.

## Gotchas

- needs a **depth map** вЂ” generate (SDF/raymarch) or accept upload.
- Dubois matrices reduce ghosting (vs naive channel split).
- lenticular "tilt" is simulated via mouse/time (screens can't physically tilt).
- wiggle is the most universally viewable.

## Ecosystem Hooks

Consumes depth from `sdf_fields`, `raymarching`, `parallax_depth_fields`. Pairs with `chromatic_aberration`, `op_art_style`.
